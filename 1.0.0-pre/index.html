Introduction

The license vendor protocol defines data schemas, protocols, and a command-line interface for offering, buying, and managing software licenses.

Software developers who build software for other developers can use tools implementing the license vendor protocol to offer paid licenses for their work.

Software developers who rely on other developersâ€™ work can use tools implementing the license vendor protocol to find out which artifacts they need licenses for, buy those licenses, and manage them over time.

Cycle

  1. A developer hosts a license vendor server, or uses one hosted by another, signs up, and configures it to sell license on their behalf.
  2. The developer adds metadata about license availability to their project, ensuring that those data will be packaged in artifacts that users download.
  3. Users who download the software run a program, such as a command-line tool, that reads the artifact's metadata and compares it against receipts for licenses the user already have.
  4. Users can run the same program to buy the licenses they need and download receipts.

Artifact Metadata

npm

Place artifact metadata in package.json under the licensevendor key.

Cargo

Place artifact metadata in Cargo.toml under [package.metadata.licensevendor].
