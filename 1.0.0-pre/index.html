Introduction

The license vendor protocol defines data schemas, protocols, and a command-line interface for offering, vending, and managing software licenses.

Developers who build software for other developers can use tools implementing the license vendor protocol to offer paid licenses for their work.

Developers who rely on other developers’ work can use tools implementing the license vendor protocol to find out which artifacts they need licenses for, buy those licenses, and manage them over time.

Overview

  1. A developer hosts a license vendor server, or uses one hosted by someone else, and configures it to vend licenses for a project on their behalf.

  2. The developer adds metadata about licenses for sale to that project, ensuring those metadata will be packaged in artifacts that users receive.

  3. Users who download the software run a client program, such as a command-line tool, that identifies artifacts with license vendor metadata and compares them against receipts for licenses that the user already has.

  4. Users run the same program to buy the licenses they need and download receipts.

Receipts

Receipts store data about licenses that software users have. Client applications use receipts to determine which licenses a user already has, and which licenses they still need.
